# Creative Companion Visual Design Guide

Creative Companion should feel like a quiet creative studio: safe enough for private reflection, structured enough to be trustworthy, and open enough for personal choice. The visual system should support emotional steadiness without becoming clinical, decorative, childish, or sentimental.

## Design Principles

- Calm, not clinical.
- Creative, not childish.
- Warm, not sentimental.
- Evidence-aware, not academic.
- Choice-centered, not directive.
- Personal, but not overly intimate.
- Supportive of low energy, uncertainty, and different comfort levels.

## Visual Direction

The interface should feel like a private desk or studio table rather than a medical portal. Visual details can suggest paper, ink, color swatches, soft studio light, and personal artifacts, but they should stay subtle enough that text remains easy to read.

Use visual elements to reduce cognitive load:

- Group choices into clear areas.
- Keep primary actions visible and calm.
- Make optional actions feel optional.
- Avoid crowding sensitive pages with too many cards or decorative elements.
- Use whitespace as emotional breathing room.

## Color And Materials

Preferred palette direction:

- Warm off-white or soft white backgrounds.
- Muted blue, green, and slate for structure.
- Soft coral, muted rose, or warm yellow as small accents.
- Gentle paper or canvas texture only if it does not reduce readability.

Avoid:

- Medical blue-and-white sterility.
- Heavy purple gradients.
- Overly cheerful rainbow palettes.
- High-alert red except for true urgent safety language.
- Dark, dramatic, or trauma-coded imagery.

## Page-Level Direction

### Home

The public home page should communicate safety, creativity, and consent quickly. It can feel more visual than a plain feature list, but it should not become a marketing page. The first screen should still make it clear what the app does and how to sign in or create an account.

Possible visual language:

- A soft studio desk composition.
- Small color swatches or paper-like panels.
- Feature entries that feel like gentle doors into the app.

### Dashboard

The signed-in dashboard should feel like a personal workspace. It should help users choose what feels useful today rather than pushing a workflow.

Possible structure:

- A calm "today" area.
- Clear entries for diary, recommendations, artwork, profile, and consent.
- Small status cues without overwhelming metrics.

### Diary

The diary should feel low-pressure and private. Emotion labels and intensity controls should be visual enough to reduce typing burden, while written reflection remains optional and spacious.

Possible structure:

- Soft emotion chips.
- A clear intensity slider.
- Recent entries and summaries separated from the writing area.
- History behind an optional view so the page does not feel crowded.

### Recommendations

The recommendation workspace is the core experience. It should feel like receiving two possible creative invitations, then choosing how to create.

Possible structure:

- Two recommendation cards with distinct visual identities.
- A "why this may fit" section that is short and humble.
- Learn-more evidence hidden behind gentle disclosure controls.
- Drawing or artwork options kept close to the recommendation so users can create while reading.

### Artwork And Gallery

The artwork area should feel like a private studio and gallery. The canvas tools can be powerful, but the first experience should not feel technical.

Possible structure:

- Simple mode by default: brush, color, undo, save.
- Studio mode for layers, shapes, opacity, and advanced tools.
- Gallery as a private artwork wall with soft frames and reflections.

### Consent And Safety

Consent and safety pages should be the clearest and least decorative. The visual tone should communicate control, transparency, and steadiness.

Possible structure:

- Plain sections.
- Clear toggles and explanations.
- No emotionally suggestive imagery.
- Crisis resources visible, calm, and direct.

## Interaction And Motion

Motion should be subtle and never attention-seeking.

Use:

- Gentle fades.
- Small transitions for opening learn-more sections.
- Clear hover/focus states.

Avoid:

- Bouncy animation.
- Surprise motion.
- Confetti, celebration effects, or gamified streaks.
- Animations that imply emotional progress or success.

## Accessibility

The visual system must support users who may be tired, distracted, in treatment, using mobile devices, or reading in a second language.

Requirements:

- Strong text contrast.
- Large enough tap targets.
- No text hidden only behind icons.
- No essential information conveyed by color alone.
- Responsive layouts that avoid overlap.
- Calm empty states that explain next steps without pressure.

## Evidence And Trust

Evidence should be visually available without overwhelming users.

Use:

- "Learn more" or "Why this may fit" sections.
- Short source summaries in plain language.
- Optional links to research notes for users who want depth.

Avoid:

- Showing many academic sources by default.
- Using citations to make recommendations sound certain.
- Presenting sources as proof that a specific activity will work.

## Implementation Sequence

1. Establish this visual direction and keep it aligned with the tone guide.
2. Redesign the dashboard and recommendation workspace first.
3. Improve diary and gallery layouts next.
4. Refine canvas controls after the broader workspace is calmer.
5. Revisit public home, consent, and safety pages for consistency.

## Review Questions

Before shipping a UI change, ask:

- Does this reduce or increase emotional pressure?
- Does it preserve user choice?
- Does it look trustworthy without looking clinical?
- Does it make the action clearer?
- Does it avoid implying diagnosis, treatment, or guaranteed benefit?
- Would this still feel respectful on a hard day?
